Sanky Diagram¶

  • Descripci: els diagrames Sanky mostren la relaci entre diferents variables o categories, i la quantitat de flux que passa entre elles. Sn grfics de xarxes que mostren la direcci del flux i la seva amplada representa la quantitat de flux que hi passa. Els diagrames Sanky sn especialment tils per representar dades sobre la relaci entre diferents categories i el flux que hi passa, i poden ser utilitzats per a qualsevol variable o categoria.
  • Exemples d'aplicaci: trfic web (visites a les pgines web), contaminacions de pasos, etc.
  • Tipus de dades: aquesta tcnica s adequada per a dades quantitatives i tamb pot ser utilitzada per a dades categriques, ja que mostra la relaci entre categories.
  • Limitacions: els diagrames Sanky sn adequats per a conjunts de dades mitjans o grans, ja que si la quantitat de dades s petita, el diagrama pot resultar poc informatiu o difcil de llegir.

# seleccionem i filtrem les variables que ens interessa graficar
df = pd.read_csv('Methane_final.csv', encoding = "UTF-8")
df = df[['region', 'type', 'emissions', 'baseYear']]
df = df.groupby(['region', 'type', 'baseYear'], as_index=False).sum()
categorias = df['region'].unique().tolist() + df['type'].unique().tolist() + df['baseYear'].unique().tolist()

# crear conexions entre les categores anteriors seleccionades
conexiones = []
for i in df.index:
    conexion = {
        'source': categorias.index(df['region'][i]),
        'target': categorias.index(df['type'][i]),
        'value': df['emissions'][i],
        'baseYear': df['baseYear'][i]
    }
    conexiones.append(conexion)

# crearem un DataFrame per les categoras i l'altre per a les conexions
df_cat = pd.DataFrame(categorias)
df_cat.columns = ['categoria']
df_con = pd.DataFrame(conexiones)

# Montem la figura del diagrama
fig = go.Figure(data=[go.Sankey(
    node = dict(
        pad = 15,
        thickness = 20,
        line = dict(color = "black", width = 0.5),
        label = df_cat['categoria'].tolist(),
    ),
    link = dict( # customitzem els labels dels fluxos
        source = df_con['source'].tolist(),
        target = df_con['target'].tolist(),
        value = df_con['value'].tolist(),
        label = df_con['value'].tolist(),
        customdata = df_con['baseYear'].tolist(),
        hovertemplate = 'De: %{source.label}<br>A: %{target.label}<br>Value: %{value}<br>Any: %{customdata}'
    ))])

# la distribuci de les emissions de CO2 per sector i regi
fig.update_layout(title="Distribuci de les emissions de Met (CH) per sector i regi, classificat per any (2019-2022) d'emissi",
                  font=dict(size=12, color='black'), 
                  height=800, width=1000)
# Subtitle de la font
fig.add_annotation(
    xref="paper",
    yref="paper",
    x=0.005,
    y=-0.05,
    text="Font: Kaggle - Dataset - Global Methane Emissions",
    showarrow=False,
    font=dict(size=12),
)
fig.show(renderer='notebook')

Font: https://www.kaggle.com/datasets/ashishraut64/global-methane-emissions?resource=download

Valoraci:

  1. A partir del conjunt de dades de les emissions globals del Met en les diferentes industries, podem extreure una visualitzaci grfica amb el diagrama Sankey on mostrem com es distribueixen les emissions de met(CH) del 2019, 2021 i 2022 entre diferents sectors en diferents regions del mn.

  2. El diagrama consisteix en un conjunt de rectangles o blocs que representen les regions i els sectors, connectats per fletxes que mostren el flux d'emissions entre ells. La mida de cada bloc i la seva posici en el diagrama depn de la quantitat d'emissions en cada regi i sector, mentre que l'amplada de les fletxes indica la quantitat d'emissions que es mouen entre ells.

  3. Ens ha sigut til per entendre com les emissions de CO2 es distribueixen entre diferents sectors i regions, i per identificar els principals focus d'emissions i les relacions entre ells. Aix pot ajudar a identificar les rees on cal reduir les emissions per complir amb els objectius de mitigaci del canvi climtic, i tamb pot ser til per a l'avaluaci de poltiques i mesures de mitigaci.

Waterfall Chart¶

  • Descripci: els grfics de cascada mostren la distribuci dels valors d'una variable a travs del temps o de diferents fases d'un procs. El grfic comena amb un valor inicial, que es representa per una barra, i desprs es mostren les diferents fases o factors que van afectant aquest valor, tant positius com negatius. Els valors s'apilen uns damunt dels altres, formant una estructura similar a la de les cascades. Aquest tipus de grfic s ideal per a la comparaci entre diferents fases o perodes.
  • Exemples d'aplicaci: finances (ingressos i despeses en diferents fases), rendiment d'una empresa (anlisi de beneficis i prdues), etc.
  • Tipus de dades: s adequada per a dades quantitatives i tamb pot ser utilitzada per a dades categriques, ja que mostra la distribuci dels valors.
  • Limitacions: els grfics de cascada sn adequats per a conjunts de dades mitjans o grans, ja que si la quantitat de dades s petita, el grfic pot resultar poc informatiu o difcil de llegir.

Rain cloud Plots¶

  • Descripci: els grfics de nvol de pluja mostren la distribuci de les dades a travs de diferents grups o categories. Aquesta tcnica combina les funcionalitats dels boxplots, les densities plots i les dot plots. Els nvols de pluja mostren les dades en forma de nvol, on la seva densitat representa la freqncia relativa de les dades. A ms a ms, tamb mostren les dades en forma de boxplots, que mostren la mediana, quartils i els valors mnim i mxim, i en forma de dot plots, que mostren cada valor individual. Aix fa que els nvols de pluja siguin una eina til per mostrar la distribuci de les dades d'una forma completa i clara.
  • Exemples d'aplicaci: al camp de la psicologia (anlisi de dades de conductes o emocions), medicina (anlisi de dades clniques), etc.
  • Tipus de dades: s adequada per a dades quantitatives, ja que mostra la distribuci de les dades. Tamb pot ser utilitzada per a dades categriques, per en aquest cas, les dades s'han de transformar en valors numrics per poder ser representades.
  • Limitacions: els nvols de pluja sn adequats per a conjunts de dades mitjans o grans, ja que si la quantitat de dades s petita, el grfic pot resultar poc informatiu o difcil de llegir. Tamb pot ser difcil interpretar els nvols de pluja si les distribucions de les dades sn molt diferents entre els grups o categories.

dades <- read.csv("2022_IRIS_Peticions_Ciutadanes_OpenData.csv", encoding = "UTF-8")
dades <- dades %>% filter(TIPUS  %in% c("INCIDENCIA") & AREA %in% c("Sanitat i salut pblica", "Transports pblics"))

# Crear variables de temps de resoluci per cada incidncia
dades$DIA_DATA_ALTA <- as.Date(dades$DIA_DATA_ALTA, format = "%d/%m/%Y")
dades$DIA_DATA_TANCAMENT <- as.Date(dades$DIA_DATA_TANCAMENT, format = "%d/%m/%Y")
dades$TEMPS_RESOLUCIO <- abs(difftime(dades$DIA_DATA_TANCAMENT, dades$DIA_DATA_ALTA, units = "days"))


# generem el plot Rainy Cloud
ggplot(dades, aes(y = TEMPS_RESOLUCIO, x = AREA, fill = AREA)) + 
    geom_flat_violin(position = position_nudge(x = .2, y = 0)) +
    coord_flip() + 
    geom_jitter(alpha = 0.5,width = 0.15) +
    theme(legend.position = "none") +
    geom_boxplot( width = .25,  outlier.shape = NA ) +
    labs(title = "Temps de resoluci d'incidncies per rea",
         y = "Temps de resoluci (dies)",
         x = "rea",
         subtitle = "Font: 2022_IRIS_Peticions_Ciutadanes_OpenData.csv") +
    scale_fill_manual(values = c("Sanitat i salut pblica" = "#0072B2", "Transports pblics" = "#009E73"))
